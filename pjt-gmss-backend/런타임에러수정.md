# 런타임 에러 수정 완료

## 🐛 발생한 에러

```
Table [bbs_risk] contains physical column name [created_at] 
referred to by multiple logical column names: [created_at], [createdAt]
```

## 🔍 원인 분석

**문제:** 엔티티 필드명이 카멜케이스 `createdAt`인데, `@Column(name = "created_at")`로 매핑하면서 Hibernate가 중복 매핑으로 인식

### 중복 매핑이 발생한 이유:
1. 필드명: `createdAt` (카멜케이스)
2. `@Column(name = "created_at")` 어노테이션
3. Hibernate가 카멜케이스를 자동으로 언더스코어로 변환 시도
4. 결과: `created_at` 컬럼에 두 개의 논리적 이름이 매핑됨

## ✅ 수정 내용

### 1. BbsRisk.java
```java
// 수정 전
@CreationTimestamp
@Column(name = "created_at")
private LocalDateTime createdAt;

@UpdateTimestamp
@Column(name = "updated_at")
private LocalDateTime updatedAt;

// 수정 후
@CreationTimestamp
@Column(name = "created_at")
private LocalDateTime created_at;

@UpdateTimestamp
@Column(name = "updated_at")
private LocalDateTime updated_at;
```
