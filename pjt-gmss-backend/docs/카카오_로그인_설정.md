# 카카오 로그인 설정 (OAuth2)

## 400 Bad Request (kauth.kakao.com/oauth/authorize)

Spring Security 6는 기본적으로 PKCE(`code_challenge`, `code_challenge_method`)를 붙여서 인가 요청을 보냅니다. 카카오 인가 코드 요청 API는 이 파라미터를 지원하지 않거나 잘못 전달 시 **400**을 반환할 수 있습니다.

- **조치**: 프로젝트에 `KakaoAuthorizationRequestResolver`를 두어, **kakao** registration에 한해 `code_challenge`, `code_challenge_method`, `code_verifier`를 제거한 요청만 보내도록 했습니다.
- `CustomSecurityConfig`에서 `oauth2Login().authorizationEndpoint().authorizationRequestResolver(kakaoAuthorizationRequestResolver)` 로 해당 리졸버를 사용합니다.

그래도 400이 나오면 **Redirect URI 미등록(KOE006)** 가능성이 있으므로, 아래 Redirect URI 설정을 다시 확인하세요.

## Redirect URI 반드시 백엔드(API)로

- **카카오 로그인이 안 되는 이유**: 콜백(redirect_uri)이 **프론트(www.gmss.site)** 로 설정되어 있으면, 카카오 인증 후 사용자가 www로 이동합니다. 그 주소에는 Spring이 없어서 **인증 코드(code)를 토큰으로 바꾸는 처리가 되지 않습니다.**
- Spring Security OAuth2는 **콜백 요청이 백엔드(API 서버)로 와야** 코드 교환·로그인 처리가 가능합니다.

## 카카오 개발자 콘솔 설정

1. [Kakao Developers](https://developers.kakao.com) → 앱 선택 → **카카오 로그인** → **Redirect URI**.
2. **등록/수정할 URL** (프로덕션):
   ```text
   https://api.gmss.site/login/oauth2/code/kakao
   ```
3. `https://www.gmss.site/login/oauth2/code/kakao` 는 **사용하지 마세요.** (프론트 URL이라 로그인 불가)
4. 로컬 개발 시에는 예:
   ```text
   http://localhost:8080/login/oauth2/code/kakao
   ```
   를 추가하고, `application.properties`(또는 로컬 프로파일)에서 `redirect-uri`를 이 주소로 맞추면 됩니다.

## 현재 백엔드 설정 (application.properties)

- `spring.security.oauth2.client.registration.kakao.redirect-uri=https://api.gmss.site/login/oauth2/code/kakao`
- 카카오 콘솔의 Redirect URI와 **완전히 동일**해야 합니다.

## 로그인 흐름 요약

1. 사용자: www.gmss.site에서 "카카오 로그인" 클릭
2. 프론트: `api.gmss.site/oauth2/authorization/kakao` 로 이동
3. Spring: 카카오 로그인 페이지로 리다이렉트 (redirect_uri=`https://api.gmss.site/login/oauth2/code/kakao` 포함)
4. 사용자: 카카오에서 로그인
5. 카카오: `https://api.gmss.site/login/oauth2/code/kakao?code=xxx` 로 리다이렉트
6. Spring: code로 토큰 발급·회원 조회·JWT 발급 후 `https://www.gmss.site/member/kakao-additional` 등으로 리다이렉트
